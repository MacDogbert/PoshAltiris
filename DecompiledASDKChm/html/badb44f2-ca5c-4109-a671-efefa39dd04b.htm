Programming with the Deployment Solution ASDK
Send Feedback

The ASDK contains components that allow you to interact with the SMP Deployment Solution (DS). In many ways the DS ASDK is designed just like the other ASDK components, such as the NS ASDK and the Software Delivery ASDK. Like the other ASDK components, you can access the DS ASDK through the COM layer, through the web service layer, or through the Command Line (CLI) layer. And like the other ASDK components, each of these layers exposes the API set through which you can perform and automate many tasks in the SMP.

However, in terms of its API design, the DS ASDK differs from other ASDK components. The other ASDK components each have a defined set of public APIs. Each of these APIs allows you to perform some specific work in, or retrieve some specific date from, the SMP. In the case of the ASDK components for the core Notification Server, for example, there are several dozen APIs. Each API represents a different discrete task.

The DS ASDK component, however, is different. Unlike the other ASDK components, the DS ASDK component exoses only a single API, called RunDeploymentSolutionTask(String). This API takes a singe argument, a string representing an XML fragment. This XML fragment follows a schema which allows you to both (1) define the task you want to perform in DS and (2) pass in all the necessary and optional data inputs that the task may need. This makes the DS ASDK highly flexible and extensible.

The DS ASDK was designed in this way because of the large and varying number and type of inputs that a DS task might require. Furthermore, collections of data about resources such as computers may need to be passed into a single task, without disrupting the relations that subsets of the data have with other subsets. For programs running in C# or Java, it might make sense to create a separate API for each task and handle the complex inputs using overloaded methods, complex data types that can store mappings between key-value pairs, and the like. However, as experienced users of the ASDK are aware, the unique nature of the ASDK's multiple-interface design precludes those kinds of approaches. Because the ASDK must support COM, web services and CLI, the scope and type of inputs into the ASDK task are limited. The solution in the case of the DS ASDK is to provide a single API that allows you to pass in complex data and data mappings through the use of XML, rather than through multiple parameters and data types.

For detailed information about the work that you can perform through the DS ASDK, as well as special COM objects that you can use to create input XML and read output XML, please see this page.
