Using Returned Values from ASDK COM Methods
Send Feedback

The Class Reference section of the ASDK documentation describes the return values for each API.

For some of these return values, additional information about Guids, "Details" classes, and DataTables (Recordsets) may be useful and is presented on this page.

Guids

The COM layer translates a Guid into a string and a Guid array into a string array.

"Details" classes

A Details class is a class used to store data fields containing information about an object within the NS. (For those versed in C/C++, it is similar in purpose to a struct.) An ASDK component library may include one or more Details classes. Each Details class is named by prepending the name of the object about which the class contains data.

For example, the ItemDetails class contains a number of fields describing a Notification Server (NS) Item. The different fields are listed in the ItemDetails Fields section.

To access a Details field, simply call that field from the object. For example, after calling CloneItem(), you can print the fields of the ItemDetails object named clonedObject as follows:

CopyVBScript
wscript.Echo "Name: " & clonedObject.Name
wscript.Echo "Guid: " & clonedObject.Guid
wscript.Echo "ParentFolderGuid: " & clonedObject.ParentFolderGuid
CopyJavaScript
WScript.echo ("Name: " + clonedObject.Name );
WScript.echo ("Guid: " + clonedObject.Guid );
WScript.echo ("ParentFolderGuid: " + clonedObject.ParentFolderGuid );

It is common to use details about an item in subsequent calls, such as:

CopyVBScript
call itemManagement.RenameItem(clonedObject.Guid, "New Name")
CopyJavaScript
itemManagement.RenameItem(clonedObject.Guid, "New Name");

DataTable (Recordset)

A few APIs return a DataTable. The DataTable is only used in the highest layer of the ASDK. COM does not support the DataTable type; therefore, in the COM layer, the DataTable is translated into a Recordset. A more complete description of Recordsets and how to use them is provided in the DataTable Conversions section of the documentation.
