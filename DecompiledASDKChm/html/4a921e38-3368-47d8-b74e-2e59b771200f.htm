The Administrator Software Development Kit (ASDK) is a set of application programming interfaces (APIs) that access the functionality of Notification Server (NS), Task Server and various NS solutions.

Who Uses It?

The ASDK’s target users are the IT personnel of NS customers, such as IT administrators (not solution developers), who want to automate or customize the funtionality of NS. Typically, ASDK users write a script, a program, or custom UI that aggregates ASDK functionality to perform desired tasks within NS.

How Is It Built?

The ASDK is built on the Symantec Management Platform, which provides the framework for various Symantec products and components, including Notification Server, Task Server, Software Delivery, and Deployment Solution. The ASDK acts as a layer between these various NS solutions and the programs that IT administrators write to access NS functionality. This layer, shown in the graphic as a green band connected to several libraries, provides simplified and/or job-based APIs that are easier for IT administrators to use than the core assemblies upon which NS and its solutions are directly built.

Touching the ASDK is a set of three programming interfaces, catering to the diversity of administrative programmers. These three interfaces (often called layers) are a Web service, COM, and a command-line interface (CLI). As shown in the graphic below, the Web service and CLI layers make calls directly into the core ASDK. The COM layer typically makes calls through the Web service layer, although most components have been constructed to allow the COM layer to also access the core ASDK directly. Each of the Web service, COM, and CLI layers are simply wrappers of the methods available in the core ASDK. This architecture conveniently concentrates functionality in a single place while offering a variety of access points.

What Is It Used For?

The ASDK is used to enhance NS usability (not to develop new products). Its purpose is to enable IT administrators to automate or customize NS functionality.

The APIs contained in the ASDK are designed to model common tasks that the IT administrator could otherwise perform through the Symantec Management Console. For example, the ASDK contains APIs for cloning, deleting, and importing items; for running reports; for managing resource items; for managing "scoping" objects; for managing Hierarchy rules and objects; for retrieving and running Site Server tasks (tasks and package services); and much more.

These APIs can be installed on NS or on any computer on which Microsoft .NET Framework v4.1 has been installed. As noted, they can be called via Web services, COM, or a command line interface (CLI). A common scenario is to install the ASDK on a remote machine and then write scripts (in the VBScript language) that call the ASDK's COM objects. The ASDK methods handle internally the data type conversions necessary to return values and objects that VBScript can then process further. Briefly stated, the ASDK allows IT administrators of Symantec Management Platform customers to customize and automate common tasks.

In this regard, it is important to distinguish the role of the ASDK from that of a more "traditional" SDK, such as the Symantec Solution Software Development Kit (SSDK). The SSDK contains a number of .NET assemblies and tools that solution developers use to write new solutions for NS. Its purpose is to facilitate the writing of new solutions that integrate into NS, whether those solutions are written by Symantec or by third-party solution developers.
