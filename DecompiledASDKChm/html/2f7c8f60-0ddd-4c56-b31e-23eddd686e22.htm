Introduction to Installing the ASDK

This page provides an overview of the ASDK installation. After you become familiar with the installation and configuration process for the type of development you want to do, continue to Installating the ASDK for instructions.

The installation process depends on the interfaces you want to use and the products you want to automate. The ASDK currently provides three development interfaces:

  • Web services, used by C# programs
  • Component Object Model (COM), typically used by VBScript
  • Command-line executables (CLI), run on the Windows command line

These interfaces are provided for each server-side library module released for the current version of the ASDK. For example, ASDK 8.0 contains modules for Notification Server and Task Server functionality. Additionally, a command-line executable is provided that allows you to run the existing COM interfaces installed with the Symantec Management Agent. The following sections provide a brief overview of the installation process according to development type.

Web Service (C#) Development

To enable C# development, you need only the documentation and samples on your development workstation. Before you begin development, you must install the server component on your Notification Server (NS). This installs the Web service you contact during application development. See the C# Programming Guide for additional information.

VBScript Development

To enable VBScript development, you need to install the COM interfaces for the products you want to program against. These components are required to execute applications during the development process.

To enable VBScript execution, the ASDK server component for the product you want to program against must be installed on NS. Additionally, the COM interface must be installed on any computers that will run the applications you create, including the server (NS) if you intend to execute applications directly on the server.

The COM interface for the Symantec Management Agent is provided by the agent installation, so no additional software is needed to write VBScript applications against the Agent. See the COM Programming Guide for additional information.

Command-Line Interface Development

To enable CLI execution for the server interfaces, the ASDK server component for the product you want to target must be installed on NS (the server). To enable the Symantec Management Agent command-line interface, the agent CLI component must be installed on any computer that will run agent command-line applications. See the CLI Programming Guide for additional information.
