Additional details on Expressions.

What are expressions?

Expressions are inventory rules of very specific types which are applied to Software Management Framework operations. The CreateExpressionXXX methods give you the ability to create expressions of the type indicated in the CreateExpression<type> methods name.

What a typical expression may look like.

CopyXML
<ruleTemplate ruleEngineCLSID="df407668-ec52-4582-8c1d-0c4d73e9530e" ruleGUID="d54675a4-ac5c-44a6-bf9f-ef63cb9dd142">
    <ruleXML>
        <!--following node(s) contains exported XML that can be fixed by user-->
        <ruleset type="Smart">
            <Prereqs></Prereqs>
            <and>
            <rule engine="{df407668-ec52-4582-8c1d-0c4d73e9530e}">
                  <SmartMSIProductCode remediate="Report" retrys="3" packageGuid="{b68d7f5b-17d9-4937-9790-83249bf28da2}" 
                  custom="False" 
                  productCode="{85f4cbcb-9bbc-4b50-a7d8-e1106771498d}"
                  SIFFile="{b68d7f5b-17d9-4937-9790-83249bf28da2}" />
                  </SmartMSIProductCode>
            </rule>
            </and>
        </ruleset>
    </ruleXML>
</ruleTemplate>

Operators

There are operators "and", "or", and "not". These operators give you the ability to modify an expression which follows these operator rules. Methods like AddOperatorToExpression will add the additional rule into the <ruleset> such as the following:

CopyXML
<and>
<rule engine="{df407668-ec52-4582-8c1d-0c4d73e9530e}">
      <SmartMSIProductCode remediate="Report" retrys="3" packageGuid="{b68d7f5b-17d9-4937-9790-83249bf28da2}" 
      custom="False" 
      productCode="{85f4cbcb-9bbc-4b50-a7d8-e1106771498d}"
      SIFFile="{b68d7f5b-17d9-4937-9790-83249bf28da2}" />
      </SmartMSIProductCode>
</rule>
</and>

The method RemoveExpression will remove such sections.

Note:
As you add sections, an ExpressionDetails structure will be returned. This structure contains a Guid member which references that specific operator modification. If you intend on removing the same operation then you must maintain a reference to its guid. Otherwise you will not have the ability to remove a particular operator from the expression.
