Installing the ASDK

Overview

A typical installation of the Administrator SDK "ASDK") involves a two-part process. The first part is the installation of the ASDK components onto the machine hosting Notification Server (NS). The second part is the optional (but recommended) installation of the COM components on any remote (non-NS) computers that meet certain requirements (see Part II, below).

This process can be summarized as follows:

  • The Symantec Installation Manager (SIM) is used to install all ASDK components onto an NS server. The ASDK is automatically installed as part of the Symantec Management Platform (SMP). It is not a separately listed installation item in SIM.
  • If desired, you can manually copy the *.msi file representing the ASDK's COM components to one or more remote computers on which you want to perform COM-based development (such as writing scripts using VBScript). Then, you manually run the MSI files to install the COM components onto the remote machine. In addition, you can manually copy another MSI file that will install a command-line interface (CLI) executable which interacts with the Symantec Agent's existing COM components (if the Agent has already been installed on the remote machine). The MSI files are located in the folder "%ProgramFiles%\Altiris\Symantec Installation Manager\Installs\Altiris". The typical command line for installing an MSI is the following: msiexec -i Symantec_asdkcomcombine_8_0_x64.msi skipaim=true , command line for uninstall MSI: msiexec -x Symantec_asdkcomcombine_8_0_x64.msi.
  • SIM installs programming example files and help files onto the NS server. These files can also be installed onto a remote development computer by copying and running the applicable MSI file.

The following table displays the various MSI installation files installed as the ASDK, with descriptions of each module and the manner of its installation. You will not manually run any of these MSI files during the ASDK installation using SIM; however, you may want to manually run the COM installation files on a remote machine if you choose to use the remote COM installation procedure described in Part II, below. In addition, it is good to have a general idea of what each component installs in the event you need to repair or reinstall certain files after your initial installation.

Installation files that have "_x86" in their filename target computers with a 32-bit OS; files that have "_64" in their name target computers with a 64-bit OS. As of SMP 7.7 32 bit is no longer supported on the SMP server, but the "_x86" COM files can still be copied and used on 32 bit operating systems.

After you have installed the ASDK components onto an NS server, the directories under the Altiris ASDK directory at the root of the installation directory will include the following:

Bin Contains command-line interface (CLI) executables.
COM Assemblies Contains COM assembly files.
Config Contains configuration files.
Web Contains the files for the ASDK web service virtual directory.
Samples Contains working example scripts for many APIs and common tasks.
Help Contains help files documentation in CHM format.

Part I: Installing ASDK Components on NS Using SIM

  1. Run SIM on the Notification Server that you want to target.
  2. Choose the platform (SMP) or any solution suite that installs the platform. The ASDK is installed automatically as part of the platform. SIM will run the installation wizard that will guide you through the installation process. SIM installs the integration assemblies, the Web services, the COM interfaces, and the command-line interface (CLI) executables for the ASDK's Server components.

You can also immediately program using VBScript or perform other COM-based programming directly on the NS server. However, to install COM components on a remote computer, continue to Part II.

Part II: Installing COM on Remote Computers for VBScript Development

Some ASDK users prefer to program using VBScript, either on the NS server or on a remote computer. You can do this by accessing the applicable COM component for the product that you want to program against.

The COM MSI files are located in the folder: "%ProgramFiles%\Altiris\Symantec Installation Manager\Installs\Altiris". The typical command line for installing an MSI is the following: msiexec /i Symantec_asdkcomcombine_8_0_x64.msi skipaim=true

To write VBScripts from a remote (non-NS) machine, the remote computer does not have to have the Symantec Management Agent installed. However, the remote machine must meet these prerequisites:

  1. Be a Windows computer that has .NET Framework v4.1 installed.
  2. Be able to access the ASDK Web services that are installed on the NS server.

You can install the COM components on any number of remote computers that meet the above requirements.

To install the COM components on a remote computer use the following procedure:

  1. On the NS server, locate the MSI file(s) for the COM component that you want to install remotely by navigating to the %ProgramFiles%\Altiris\Symantec Installation Manager\Installs\Altiris directory.
  2. Copy the MSI to the remote computer that you want to install them on.
    For 64-bit computers, copy the Symantec_asdkcomcombine_8_0_x64.msi file.
    For 32-bit computers, copy the symantec_asdkcomcombine_8_0_x86.msi file.
  3. Use the following command line to install the COM file:
    msiexec /i Symantec_asdkcomcombine_8_0_x64.msi skipaim=true.

Note:
All prerequisites for the remote computer are identical to those indicated above for the normal COM installation process.
