Using a URL Query String
Send Feedback

To access an ASDK Web service via your browswer, append a command name and query string to the end of the Web service URL in the following format:

http://<servername>/Altiris.<libraryname>/<webservicename>Service.asmx/<commandname>?<param1name>=<value1>&<param2name>=<value2> ....

For example, to run the RenameItem() method in the ItemManagement Web service from the ASDK.NS library (assembly), you would type the following into your browser's address box (changing the value of the arguments as appropriate):

http://myserver/Altiris/ASDK.NS/ItemManagementService.asmx/RenameItem?itemGuid=b535fb57-09e1-4e9f-bb13-b4600094cf61&newName=MyName

Note that the name of the command (Web service method) comes before the question-mark (?) delimiter, and not after it as you might expect. The command follows the name of the Web service and the forward slash (/), and then the question mark (?) delimiter follows the name of the command. Following the question mark are the name-value pairs representing the input parameters to the ASDK method.

Please note also that the query string format does not lend itself well to complex parameter values, such as parameters consisting of complex XML. (Complex XML can cause problems with the parsing of the query string.) Consequently, the URL query string method of accessing the ASDK Web services is more convenient for methods that take simpler parameter values such as single GUIDs and simple strings.
