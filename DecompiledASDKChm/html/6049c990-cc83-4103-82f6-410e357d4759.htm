Welcome to the "portal" page for programming with the COM layer of the Deployment Solution ASDK. The following tables summarize the various discrete units of work you can perform in Deployment Solution using the ASDK. They also list any applicable "input helper" and "output helper" classes that you can use in VBScript code (or other COM-compatible language) that you write. There are also links and details about sample input XML and sample scripts that show how to perform various tasks.

The input helper classes make it easier to construct the input XML string that the main API (RunDeploymentSolutionTask(String)) takes as its sole parameter. The output helper classes retrieve the data returned from the call to RunDeploymentSolutionTask -- which is also in the form of XML -- and parse and store the returned data into fields in the helper class. In that way, you can more easily access a piece of data returned from an earlier task, to use as input for a subsequent task.

The first table shows how you can interact with settings using the Deployment Solution ASDK. The second table lists the ways in which you can interact with resources. The third table details the various tasks that you can create.

Note:

Symantec recommends you use input helper classes to facilitate the creation of the input XML string that is to be passed into the RunDeploymentSolutionTask API. However, you can find examples of how to call RunDeploymentSolutionTask without using input helper classes -- i.e., using hand-coded XML strings directly as your input -- in the sample script [ASDK Intallation Folder]\Samples\DSTaskManagement\VBS\RunDeploymentSolutionTask.vbs.

Creating Tasks using the Deployment Solution ASDK

Caution:

The Deployment Solution ASDK creates Tasks in the Symantec Management Platform that, when executed using the SMP Task Server, will perform the actions described below. It is important to understand that the Deployment Solution ASDK only creates these tasks -- it does not run the tasks once they are created. If you want to run one of these tasks, you can call the ExecuteTask(Guid, String, String) API from the ASDK's Task Management library. Or, you can simply run the Task directly from the SMP Console.
